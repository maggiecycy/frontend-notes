
# 🛡️  (The Odin Project) - HTML 基础知识点总结

> HTML 基础部分主要侧重于**网页骨架的搭建**和**标签的语义化**。
> HTML 的核心任务是告诉浏览器“这是什么”（结构），而不是“它长什么样”（这部分交由 CSS 负责）。

---

## 1. HTML 文档结构 (Boilerplate)

这是每个规范的 HTML 网页都必须具备的标准“骨架”：

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>网页标题</title>
</head>
<body>
    </body>
</html>
```

* **`<!DOCTYPE html>`**：文档类型声明，告诉浏览器当前页面使用的是 HTML5 标准。
* **`<html>`**：网页的根元素，所有其他 HTML 元素都嵌套在其中。通常会添加 `lang` 属性（如 `<html lang="en">`）。
* **`<head>`**：包含网页的“元数据”（不在页面上直接显示的内容）。
  * **`<meta charset="UTF-8">`**：设置字符编码，确保网页能正确显示各种语言，防止乱码。
  * **`<title>`**：网页的标题，显示在浏览器的标签页上，对搜索引擎优化（SEO）很重要。
* **`<body>`**：包含网页的所有可见内容（文本、图像、链接等）。

---

## 2. 文本与语义化 (Text & Semantics)

使用正确的标签来赋予内容逻辑意义：

* **标题 (`<h1>` - `<h6>`)**：用于构建页面的大纲。`<h1>` 是最重要的主标题（每个页面通常只用一次），`<h6>` 重要性最低。
* **段落 (`<p>`)**：用于包裹普通的文本段落。
* **强调与加粗 (`<em>` 和 `<strong>`)**：
  * **`<em>`**：表示语气上的强调（默认呈现为斜体）。
  * **`<strong>`**：表示内容的重要性、严重性或紧急性（默认呈现为粗体）。

---

## 3. 列表 (Lists)

用于组织相关联的数据或项目：

* **无序列表 (`<ul>`)**：项目之间没有严格的先后顺序（默认呈现为项目符号/圆点）。
* **有序列表 (`<ol>`)**：项目之间有明确的顺序（默认呈现为数字或字母序号）。
* **列表项 (`<li>`)**：必须直接嵌套在 `<ul>` 或 `<ol>` 内部，表示列表中的具体条目。

---

## 4. 链接与导航 (Links)

使用锚点元素 **`<a>`** 创建超链接：

* **核心属性 (`href`)**：Hypertext Reference，指定链接跳转的目标地址。
* **相对路径与绝对路径**：
  * **绝对路径**：包含完整的 URL（如 `https://www.theodinproject.com`），主要用于跳转到其他网站。
  * **相对路径**：基于当前文件所在位置的路径（如 `../pages/contact.html`），用于在同一个网站项目内部互相链接。`../` 表示返回上一级父目录。
* **安全性考量**：当使用 `target="_blank"` 让链接在新标签页中打开时，**必须**加上 `rel="noopener noreferrer"` 属性。这能防止新打开的页面通过 JavaScript 劫持原网页，同时保护来源隐私。

---

## 5. 图像 (Images)

使用 **`<img>`** 元素在网页中嵌入图像。这是一个“空元素”（自闭合标签），不需要结束标签。

* **两大必填属性**：
  * **`src`** (Source)：指定图像文件的路径或 URL。
  * **`alt`** (Alternative text)：替代文本。当图片链接失效时会显示该文本；对于使用屏幕阅读器的视障用户来说，这也是理解图片内容的唯一途径。
* **常用格式**：
  * **JPEG**：适合色彩丰富的照片。
  * **PNG**：适合需要透明背景或边缘清晰的图形（如 Logo）。
  * **GIF**：适合简单的短动画。
  * **SVG/WebP**：SVG 适用于缩放不失真的矢量图；WebP 是现代高压缩比格式。
